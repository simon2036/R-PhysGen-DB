## Executed command plan

```bash
PYTHONPATH=src .venv/bin/python pipelines/generate_governance_quantum_batches.py \
  --mode xtb-pregeometry --batch-size 100

PYTHONPATH=src .venv/bin/python pipelines/run_xtb_quantum_pilot.py \
  --requests data/lake/raw/generated/governance_xtb_requests_batch001.csv \
  --xyz-manifest data/lake/raw/generated/governance_xtb_xyz_manifest_batch001.csv \
  --xtb-bin /home/biosphere/micromamba/envs/rh_round1/bin/xtb \
  --jobs 48 --threads-per-job 1 --completion-required

PYTHONPATH=src .venv/bin/python pipelines/generate_governance_quantum_batches.py \
  --mode psi4-singlepoint --batch-size 50

PYTHONPATH=src .venv/bin/python pipelines/run_psi4_quantum_pilot.py \
  --requests data/lake/raw/generated/governance_dft_singlepoint_requests_batch001.csv \
  --xyz-manifest data/lake/raw/generated/governance_dft_singlepoint_xyz_manifest_batch001.csv \
  --psi4-bin /home/biosphere/micromamba/envs/rphysgen-psi4/bin/psi4 \
  --scratch-dir data/lake/raw/manual/quantum_pilot_artifacts/_psi4_scratch \
  --jobs 16 --completion-required

PYTHONPATH=src .venv/bin/python pipelines/run_psi4_quantum_pilot.py \
  --requests data/lake/raw/generated/governance_dft_singlepoint_requests_batch002.csv \
  --xyz-manifest data/lake/raw/generated/governance_dft_singlepoint_xyz_manifest_batch002.csv \
  --psi4-bin /home/biosphere/micromamba/envs/rphysgen-psi4/bin/psi4 \
  --scratch-dir data/lake/raw/manual/quantum_pilot_artifacts/_psi4_scratch \
  --jobs 16 --completion-required
```

Retry was not needed because both DFT batches completed with zero failures. The retry policy remains: failed-only batches with `--retry-failed-only --jobs 8` when the failed count is five or fewer; stop and diagnose before downstream batches when a batch has more than five failures.

## Boundary notes

- No scalar values are fabricated by the batch generator or retry path.
- Request IDs reuse the existing xTB and Psi4 hash rules so completed jobs are skipped rather than duplicated.
- Quantum descriptors remain excluded from wide model tables.
- Unmapped governance substances stay visible in `governance_dft_mapping_report.csv` and are not silently dropped.
- Phase-2 DFT products are still intentionally absent until a separate frequency/charge/thermochemistry/conformer plan is implemented and verified.
