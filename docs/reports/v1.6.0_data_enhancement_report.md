## Scope completed

This upgrade keeps raw/manual and raw/generated inputs as the source of truth and rebuilds all Parquet/DuckDB outputs through the normal pipeline. No Parquet outputs were hand-edited.

Implemented scope:

- Promoted A/B/C coverage matrix and enrichment worklist with explicit `r_number`, `canonical_smiles`, `canonical_feature_key`, and `missing_feature_key` columns.
- Tier D PubChem candidate expansion to a 20,000-row generated candidate file. Tier D remains inventory-only and excluded from promoted strong-label readiness gates.
- Native canonical projection from high-quality `property_recommended` rows into canonical and strict canonical outputs without overwriting governance-bundle rows.
- Manual environmental evidence supplement at `data/lake/raw/manual/observations/environment_safety_20260429.csv`: 37 auditable rows, comprising 4 GWP100 rows from the German Environment Agency (UBA) GWP table and 33 ODP zero-screen rows from an EPA ODS-mechanism/formula screen.
- CoolProp cycle/transport integration retained from the v1.6 workspace; unresolved transport/cycle cases remain pending rather than fabricated.
- Quantum-pilot execution: 158 active-learning-selected requests, 158 generated XYZ inputs, 158 local xTB GFN2 jobs, and 948 ingested scalar quantum observations. No Psi4/DFT values were fabricated.
- Deterministic active-learning queue using weights `0.30 performance + 0.25 uncertainty + 0.20 novelty + 0.15 feasibility + 0.10 coverage_gap`; queue now mixes promoted gaps with Tier D candidates and routes actions by gap type.
- Readiness semantics updated so documented strict-label warnings do not become hard failures when must-have coverage and aggregate should-have coverage pass.
