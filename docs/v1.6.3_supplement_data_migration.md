# v1.6.3 Supplement Data Migration

Date: 2026-05-05
Target dataset version: `v1.6.3-draft`

## Scope

This data-only migration integrates the combined SupplementData curation pack dated 2026-05-01 without changing the public table schemas or the target dataset version.

Authoritative input: `SupplementData/r_physgen_curation_pack_combined_20260501.zip`. The non-combined zip is treated as a subset of the combined package.

## Integrated strict observations

- Added `data/raw/manual/observations/manual_property_observations_supplement_combined_20260501.csv`.
- The file keeps the existing 19-column manual observation contract.
- It contains 95 strict-eligible manual observations.
- Rows are loaded through the existing optional observations directory path, not appended to the legacy `manual_property_observations.csv`.
- Current identity resolution is expected to admit the resolvable subset into `property_observation`; unresolved identity rows remain raw evidence until identity curation resolves them.

## MIX_511A / R-511A curation

Supplement evidence indicates R-511A is R-290/R-E170 (95.0/5.0 mass fraction). The prior extension layer represented the second component as R-170 ethane. This migration adds:

- `data/raw/manual/mixture_component_curations.csv` to replace `mol_otmsdbzupauedd-uhfffaoysa-n` (R-170 ethane) with `mol_lcglnkutagevqw-uhfffaoysa-n` (R-E170 dimethyl ether) for `MIX_511A`.
- `data/raw/manual/mixture_fraction_curations.csv` to set R-290 to `0.95` and R-E170 to `0.05` on a mass-fraction basis.

Both curation files require traceable source metadata and are registered in `source_manifest` when loaded.

## Review-only inequality evidence

- Added `data/raw/manual/review_only/review_only_inequality_observations_round2_20260501.csv`.
- These 13 observations report inequality/bound evidence such as `<1` and `<<1`.
- They are registered in `source_manifest` for auditability but intentionally kept outside `data/raw/manual/observations/`, so they do not enter `property_observation` or model-ready layers under the current strict numeric policy.

## Schema impact

No breaking schema change. `build_mixture_tables(...)` now accepts an optional `component_curations` keyword argument, and the mixture composition contract documents the optional source id used by manual component curations.
