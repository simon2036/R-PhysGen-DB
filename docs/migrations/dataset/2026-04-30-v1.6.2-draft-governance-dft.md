# Dataset Migration: v1.6.2-draft governance DFT batching

## Summary

This migration upgrades the draft dataset from `v1.6.1-draft` to `v1.6.2-draft` for governance-specific high-compute DFT scheduling and execution.

The change adds `pipelines/generate_governance_quantum_batches.py`, which materializes the `tbl_qm_dft_required_queue` backlog into auditable xTB pregeometry and Psi4/DFT singlepoint batches without fabricating quantum values. The first-stage DFT scope remains limited to scalar features already supported by the canonical quantum pilot path: total energy, HOMO, LUMO, HOMO-LUMO gap, dipole moment, and polarizability. Frequency/ZPE/IR, NBO/RESP charges, thermochemistry, and conformer ensembles remain explicitly deferred.

## Compatibility

The migration is additive. Existing table contracts are unchanged, and quantum descriptors remain long-form observations plus `quantum_job`/`quantum_artifact` audit records. This migration does not add quantum wide columns to `property_matrix`, `model_ready`, or `model_dataset_index`.

New raw/generated planning artifacts include:

- `data/lake/raw/generated/governance_xtb_requests_batch*.csv`
- `data/lake/raw/generated/governance_xtb_xyz_manifest_batch*.csv`
- `data/lake/raw/generated/governance_xtb_xyz/*.xyz`
- `data/lake/raw/generated/governance_dft_singlepoint_requests_batch*.csv`
- `data/lake/raw/generated/governance_dft_singlepoint_xyz_manifest_batch*.csv`
- `data/lake/raw/generated/governance_dft_singlepoint_xyz/*.xyz`
- `data/lake/raw/generated/governance_dft_mapping_report.csv`
- `data/lake/raw/generated/governance_dft_mapping_summary.json`

## Governance queue state and execution result

The governance DFT backlog contains `1350` request rows: `135` substances × `10` requested high-level QM outputs. The current molecule inventory maps `62` of those substances and preserves `73` unmapped substances in `governance_dft_mapping_report.csv` with explicit mapping reasons.

Before this migration pass, `24/62` mapped governance molecules already had completed xTB coverage and `7/62` already had completed Psi4/B3LYP-D3BJ/def2-SVP coverage. This pass then executed the remaining currently supported first-stage scalar route:

- xTB pregeometry batch: `38` newly queued GFN2-xTB jobs, `38` succeeded, `0` failed.
- Psi4/DFT batch 001: `50` newly queued B3LYP-D3BJ/def2-SVP singlepoint jobs, `50` succeeded, `0` failed.
- Psi4/DFT batch 002: `5` newly queued B3LYP-D3BJ/def2-SVP singlepoint jobs, `5` succeeded, `0` failed.
- Current mapped governance coverage: `62/62` have completed xTB and `62/62` have completed Psi4/B3LYP-D3BJ/def2-SVP scalar observations.
- Current mapped governance blockers: `0` pending DFT requests and `0` missing-xTB-geometry blockers.

During execution, the Psi4 runner was corrected so batched `--completion-required` runs preserve successful unrelated Psi4 rows while still removing stale failure-audit rows for the active manifest.

## Required actions to reproduce

```bash
PYTHONPATH=src .venv/bin/python pipelines/generate_governance_quantum_batches.py \
  --mode xtb-pregeometry --batch-size 100

PYTHONPATH=src .venv/bin/python pipelines/run_xtb_quantum_pilot.py \
  --requests data/lake/raw/generated/governance_xtb_requests_batch001.csv \
  --xyz-manifest data/lake/raw/generated/governance_xtb_xyz_manifest_batch001.csv \
  --xtb-bin /home/biosphere/micromamba/envs/rh_round1/bin/xtb \
  --jobs 48 --threads-per-job 1 --completion-required

PYTHONPATH=src .venv/bin/python pipelines/generate_governance_quantum_batches.py \
  --mode psi4-singlepoint --batch-size 50

PYTHONPATH=src .venv/bin/python pipelines/run_psi4_quantum_pilot.py \
  --requests data/lake/raw/generated/governance_dft_singlepoint_requests_batch001.csv \
  --xyz-manifest data/lake/raw/generated/governance_dft_singlepoint_xyz_manifest_batch001.csv \
  --psi4-bin /home/biosphere/micromamba/envs/rphysgen-psi4/bin/psi4 \
  --scratch-dir data/lake/raw/manual/quantum_pilot_artifacts/_psi4_scratch \
  --jobs 16 --completion-required

PYTHONPATH=src .venv/bin/python pipelines/run_psi4_quantum_pilot.py \
  --requests data/lake/raw/generated/governance_dft_singlepoint_requests_batch002.csv \
  --xyz-manifest data/lake/raw/generated/governance_dft_singlepoint_xyz_manifest_batch002.csv \
  --psi4-bin /home/biosphere/micromamba/envs/rphysgen-psi4/bin/psi4 \
  --scratch-dir data/lake/raw/manual/quantum_pilot_artifacts/_psi4_scratch \
  --jobs 16 --completion-required
```

If future rematerialization produces failures, retry failed-only batches with `--retry-failed-only --jobs 8` when the failed count is five or fewer. If failures exceed five, stop downstream batches and diagnose input/convergence issues first. Failed requests must retain audit artifacts and must not be converted into scalar values.
