# Dataset Migration: v1.6.3-draft phase-2 quantum handoff

## Summary

This migration promotes the validated local phase-2 governance quantum state from the `v1.6.2-draft` governance-DFT baseline into the `v1.6.3-draft` release handoff.

The initial handoff did not rerun heavy xTB, CREST, or ORCA executors. A follow-up residual computation pass is recorded in `2026-04-30-v1.6.3-draft-residual-computation-pass.md`: it reran only the two xTB Hessian imaginary-frequency cases, reconciled completed Psi4 DFT manifest rows, and ingested the cycle backend rows available from CoolProp.

## Phase-2 execution state

The phase-2 scope adds frequency/thermochemistry and conformer summary descriptors for mapped governance molecules while preserving the long-form-only quantum contract.

- xTB Hessian requests: `62` attempted for mapped governance molecules.
- xTB Hessian acceptance: `60` molecule-level Hessian observations accepted; `2` attempts are retained as failed `quantum_job` entries because `imaginary_frequency_count=1`.
- Accepted xTB Hessian scalar rows: `240` = `60` accepted jobs × `4` features (`zpe`, `lowest_real_frequency`, `thermal_enthalpy_correction`, `thermal_gibbs_correction`).
- CREST conformer summaries: `62` succeeded, producing `124` scalar observations (`conformer_count` and `conformer_energy_window`).
- ORCA opt/freq: request and XYZ manifests were generated for smoke plus full batches, but ORCA was not executed and no ORCA rows were ingested.

## Residual computation supplement

The follow-up pass for the user's remaining-computation request produced:

- Psi4 DFT expansion manifest: `150` requests marked `completed` from existing `900` completed Psi4 scalar rows.
- xTB Hessian retry: the `2` imaginary-frequency jobs were rerun and still reproduced `imaginary_frequency_count=1`; accepted Hessian rows remain unchanged.
- Cycle backend retry: `21` `run_cycle` queue entries attempted with backend `auto`; `4` CoolProp successes wrote `16` rows to `data/lake/raw/manual/cycle_backend_results.csv`; `17` entries remain blocked on REFPROP/backend coverage.
- ORCA opt/freq remains blocked until a real ORCA quantum-chemistry binary is configured.
