# Dataset Migration: v1.6.0-draft enhancement

## Summary

This migration upgrades the file-lake dataset from `v1.5.0-draft` to `v1.6.0-draft` for the full data-enhancement pass.

The change keeps raw/manual and raw/generated inputs as the source of truth and rebuilds Parquet/DuckDB outputs through the normal pipeline. It adds promoted A/B/C coverage worklists, native canonical projection for high-quality recommended values, manual environmental evidence, CoolProp cycle/transport outputs, 158 active-learning-selected quantum-pilot request/XYZ manifests, 158 executed local xTB jobs with ingestion-ready scalar results, and a deterministic active-learning queue.

## Compatibility

The migration is additive. Existing core tables remain present. New or expanded generated artifacts include:

- `data/lake/raw/manual/observations/environment_safety_20260429.csv`
- `data/lake/raw/generated/promoted_coverage_matrix.csv`
- `data/lake/raw/generated/promoted_enrichment_worklist.csv`
- `data/lake/raw/generated/active_learning_queue.csv`
- `data/lake/raw/generated/quantum_pilot_requests.csv`
- `data/lake/raw/generated/quantum_pilot_xyz_manifest.csv`
- `data/lake/raw/generated/quantum_xyz/*.xyz`
- `data/lake/raw/manual/quantum_pilot_results.csv`
- `data/lake/raw/manual/quantum_pilot_artifacts/`
- expanded `data/lake/gold/property_recommended_canonical*.parquet`
- non-empty `data/lake/gold/active_learning_queue.parquet`
- non-empty `data/lake/silver/quantum_job.parquet` and `data/lake/silver/quantum_artifact.parquet`

Tier D remains inventory-only (`model_inclusion=no`) and is excluded from promoted A/B/C strong-label gates.

## Source-of-truth and generated-output policy

Primary source-of-truth files for this migration are the raw/manual and raw/generated CSV/XYZ inputs listed above, plus the schema and documentation changes. Parquet, JSON reports, and DuckDB files under `data/lake/bronze`, `data/lake/silver`, `data/lake/gold`, `data/lake/extensions`, and `data/indexes` are rebuild outputs from `pipelines/build_v1_dataset.py` and `pipelines/validate_v1_dataset.py`.

Local PubChem request/response JSON cache files matching `data/lake/raw/pubchem/tierd_excel202603_*.json` are local-only and ignored. Do not treat them as release source-of-truth; use `data/lake/raw/generated/pubchem_tierd_candidates.csv` instead.

## Data governance notes

- The manual environmental supplement adds 4 public GWP100 rows and 33 ODP zero-screen rows. It does not add safety-group values without exact auditable safety classifications.
- `MIX_511A` remains in the mixture gap registers; no unaudited blend composition or property value was inserted.
- Quantum execution for this migration is limited to the local xTB pilot: 158 GFN2-xTB jobs were executed from the generated XYZ inputs, producing 948 scalar long-form observations plus per-request artifact bundles.
- xTB partial charges, WBO, stdout/stderr, JSON, optimized XYZ, and related files remain traceable artifacts. They are not widened into `property_matrix`, `model_ready`, or `model_dataset_index`.
- No Psi4/DFT values were fabricated, and xTB values were not copied into higher-fidelity DFT fields.
- Active-learning queue entries are proposed work items only. They do not mutate `model_ready` or recommended property tables.
