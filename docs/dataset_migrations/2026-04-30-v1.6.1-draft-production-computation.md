# Dataset Migration: v1.6.1-draft production computation

## Summary

This migration upgrades the draft dataset from `v1.6.0-draft` to `v1.6.1-draft` for the production-computation pass.

The change preserves the v1.6.0 raw evidence and extends the computation pipeline so production builds can generate 2,000 xTB request/XYZ inputs, select 150 Psi4/DFT requests from completed xTB molecules, force-complete the xTB and Psi4 batches without counting failed calculations as complete, merge xTB/Psi4 result rows without duplicate scalar identities, and emit multi-condition CoolProp cycle observations for every explicitly mapped supported CoolProp fluid. Executor-unavailable or failed-attempt conditions remain auditable artifacts rather than fabricated scalar values.

## Compatibility

The migration is additive. Existing core tables remain present. New or expanded artifacts include:

- `data/raw/generated/quantum_pilot_requests.csv` expanded to the configured production xTB target.
- `data/raw/generated/quantum_pilot_xyz_manifest.csv` and `data/raw/generated/quantum_xyz/*.xyz` expanded with deterministic XYZ inputs.
- `data/raw/generated/quantum_dft_requests.csv` for the 150-row Psi4/DFT production queue.
- `data/raw/generated/quantum_dft_xyz_manifest.csv` and `data/raw/generated/quantum_dft_xyz/*.xyz` for DFT executor inputs, preferring completed xTB optimized geometries when available.
- `data/raw/manual/quantum_pilot_results.csv` as the unified xTB/Psi4 long-form scalar/audit result file, de-duplicated by `(request_id, program, theory_level, canonical_feature_key)`.
- `data/raw/manual/quantum_pilot_artifacts/` with per-request input, stdout/stderr, return code, result bundle, and `manifest.sha256` files for new executor attempts.
- Expanded CoolProp cycle rows in `data/silver/property_observation.parquet`, `data/silver/cycle_case.parquet`, and `data/silver/cycle_operating_point.parquet`.

Tier D remains inventory-only (`model_inclusion=no`) and is excluded from promoted A/B/C strong-label gates. Quantum features remain long-form observations and do not widen into `property_matrix`, `model_ready`, or `model_dataset_index`.

## Source-of-truth and generated-output policy

Raw/manual and raw/generated CSV/XYZ inputs plus retained quantum artifact directories are the source of truth for this migration. Parquet, JSON reports, and DuckDB files under `data/bronze`, `data/silver`, `data/gold`, `data/extensions`, and `data/index` are rebuild outputs from `pipelines/build_v1_dataset.py` and `pipelines/validate_v1_dataset.py`.

Executor absence is not a reason to fabricate data. If xTB or Psi4 is unavailable, the runner may be invoked with `--allow-missing-executor` to preserve a failed audit artifact for each request. Successful xTB/Psi4 values must come from executor output artifacts only.

## Data governance notes

- Existing GWP/safety/MIX_511A unresolved boundaries from v1.6.0 remain in force unless an auditable source is added. This migration does not insert unaudited blend composition or safety classifications.
- xTB and Psi4 rows share one ingestion CSV and are merged by calculation identity; newer success rows replace older duplicates for the same scalar and supersede blank failure audit rows for the same request/program.
- Psi4/DFT requests are selected only from molecules with completed xTB scalar coverage, using deterministic coverage/scaffold/halogen/priority stratification.
- Psi4/DFT requests use real Psi4 outputs from a dedicated conda/micromamba environment; no xTB scalar values are copied into DFT fields.
- CoolProp calculations are emitted only for explicit seed/alias mappings supported by CoolProp. Unsupported fluids remain warning/audit events.

## Verification results

Verification is recorded in `data/gold/quality_report.json`, `data/gold/validation_report.json`, and the final handoff summary for the implementing session. Expected acceptance checks for this migration are:

- `validation_report.errors == []`.
- xTB production target is 2,000 requests; final raw and silver outputs have `2000` succeeded and `0` failed jobs.
- Psi4/DFT target is 150 requests selected from completed xTB molecules; final raw and silver outputs have `150` succeeded and `0` failed jobs.
- Quantum artifacts have non-empty SHA-256 values and retain failed-attempt bundles for auditability.
- Multi-condition CoolProp cycle outputs include COP, pressure ratio, volumetric cooling capacity, and discharge temperature for resolved operating points.
- `model_ready` does not include quantum wide columns.
- Tier D remains `model_inclusion=no`.
- `MIX_511A` remains unresolved unless exact auditable evidence is added.
